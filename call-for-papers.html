Frequently Asked Questions

When c4p ends?

We do not have a hard deadline. Applications are accepted on an ongoing basis.

If we thing that your workshop is cool, you will get a confirmation of acceptance. If somebody proposed a similar workshop or we do not have enough rooms, we will have to reject you proposal.

We encourage you to enter c4p as soon as possible!

How long is the workshop?

One workshop lasts 3 hours. If you have a topic or presentation that does not fill three hours, then we can offer you 1.5 hours. We prefer, however, three-hour workshop.

Can I speak on Friday or Saturday?

We frame an agenda, when we receive enough workshops on the given subject. If you care for a specific day - mark this in c4p or contact us directly.

What is the scope of the workshops?

We are going to do 32 parallel thematic paths. Draft (which may still change depending on the supply of workshops) is as follows:

  • 2 paths iOS
  • 3 paths Android
  • 2 tracks DevOps / Cloud
  • 2 paths Terminal / Tools
  • 2 Agile Path
  • 2 tracks Scala
  • 3 Java path
  • 1 track GameDev
  • 2 tracks Groovy
  • 3 tracks Python
  • 2-track monitoring / Scalability / Performance
  • 3 Path JavaScript
  • 3 tracks Ruby
  • 2 paths Hardware

You can submit a workshop in each of the above topics.

In what language should I speak in a workshop?

You can speak in Polish or English. It would be best if you were able to run in English, if only one of the participants ask for it. However, if you do not feel up to do this in English, that's OK too. There is no difference to us, but please note the language in the description of the workshop. You do not want to unpleasantly surprise your participants.
